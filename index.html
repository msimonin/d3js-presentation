3. Loading datas

Load datas in memory

  • Hard coded :
    • 
var data = [1,2,3];
// create visualization
  • Through asynchronous HTTP request : 
    • 
d3.json("path/to/file.json", function(error, json) {
  if (error) return console.warn(error);
  data = json;
  // create visualization
});

4. Scales

4. scales

From the wiki :

Scales are functions that map from an input domain to an output range


Linear scale : 

var linScale = d3.scale.linear().domain([0,10]).range([0,1000]);
linScale(2.5); // return 250

By default: domain and range are [0,1]

4. scales

Typical use case : draw svg elements in the screen coordinate system 
var WIDTH = 500; // svg width
var HEIGHT = 100;
var datas = [1,20,10,5,30,25,40];
var xScale = d3.scale.linear().domain([0, 42]).range([0,WIDTH]);
var svg = d3.select('body')
  .append('svg')
  .attr('width', WIDTH)
  .attr("height", HEIGHT);
svg.selectAll('circle')
   .data(datas)
   .enter()
   .append('circle')
   .attr("cx", function(d){
     return xScale(d)
   .attr("cy", HEIGHT/2);
})

5. Axis

5. Axis

For a given scale, the d3 axis component generate all the svg elements needed to draw the axis (a line, ticks, numbers ...) see wiki.



var linScale=d3.scale.linear().domain([0,1]).range([0,500]);
var xAxis = d3.svg.axis().scale(linScale);
var svg = d3.select('body').append('svg').attr("width", 300).attr("height", 500);
//axis will be on the top of the screen
svg.append("g").call(xAxis);
// we often need to translate down the x axis
svg.append("g").attr("transform", "translate(0,100)").call(xAxis);
>> example

6. Interactions

Access the event 
{selection}.on('mouseover', function(d){
  // raw event
  console.log(d3.event);
  // local coordinate where the event has been fired
  console.log(d3.mouse(this));
})
d3.touch and d3.touches are available as well : https://github.com/mbostock/d3/wiki/Selections#d3_mouse
